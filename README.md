# Универсальный тестер логических микросхем 74HCxx (DIP‑14/DIP-16) на ESP32

## Описание проекта
Этот проект представляет собой универсальный тестер логических микросхем серии 74HCxx в корпусе DIP‑14.  
Тестер работает на базе ESP32 и позволяет проверять корректность работы следующих микросхем:

- 74HC00 — NAND
- 74HC02 — NOR
- 74HC08 — AND
- 74HC32 — OR
- 74HC74 — D flip-flop
- 74HC109 — JK flip-flop

Тестер использует фиксированную привязку выводов DIP‑14/DIP-16 к GPIO ESP32.  
Для каждой микросхемы задаётся её топология (какие выводы являются входами A и B, а какой вывод является выходом Y).  
Логическая функция (AND, OR, NAND, NOR) задаётся отдельно.  
Проверка триггеров (D и JK) выполняется отдельными функциями.  
Проект легко расширяется под другие микросхемы.

## Важные ограничения ESP32

Некоторые GPIO ESP32 являются только входами:

- GPIO34
- GPIO35
- GPIO36
- GPIO39

Их нельзя использовать как выходы.  
Если входы микросхемы попадают на такие пины, ESP32 выдаст ошибку:

"GPIO can only be used as input mode"  
"gpio_set_level(): GPIO output gpio_num error"

Также GPIO32 имеет ограничения и может работать нестабильно как выход.

Для микросхемы 74HC02 (NOR) это особенно важно, так как её топология отличается от NAND/AND/OR.

## Структура проекта

Проект: Универсальный тестер 74HCxx + 74HC74 + 74HC109
1. PIN[] — массив привязки DIP‑выводов к GPIO ESP32 (для 14-выводных микросхем)
2. PIN16[] — массив привязки для 16-выводных микросхем (74HC109)
3. Gate — структура, описывающая один логический элемент (A_pin, B_pin, Y_pin)
4. HC00[], HC02[] — массивы топологий микросхем (HC08, HC32 используют HC00)
5. fNAND, fNOR, fAND, fOR — логические функции
6. blinkLED() — функция индикации (RED_PIN=32, GREEN_PIN=33)
7. test_chip() — универсальная функция тестирования логических микросхем
8. test_d_flipflop() — тестирование одного D-триггера
9. test_74HC74() — тестирование всей микросхемы 74HC74
10. clkPulse() — генерация тактового импульса
11. init109() — инициализация триггера 74HC109
12. test_jk_flipflop() — тестирование одного JK-триггера
13. test_SN74HC109() — тестирование всей микросхемы SN74HC109
14. handleCommand() — обработка команд (HC00, HC08, HC32, HC02, HC74, HC109)
15. setup() — инициализация Serial, пинов LED
16. loop() — чтение команд из Serial и вызов handleCommand()

## Как работает тестирование

**Для логических микросхем (74HC00, 74HC02, 74HC08, 74HC32):**

1. Для каждого элемента микросхемы выбираются входы A и B и выход Y.
2. На входы подаются все комбинации 0 и 1.
3. С выхода считывается результат.
4. Результат сравнивается с ожидаемым значением логической функции.
5. Для каждого элемента выводится OK или FAIL.

**Для D-триггеров (74HC74):**

1. Проверяется асинхронный RESET (CLR = LOW → Q = 0, /Q = 1).
2. Проверяется асинхронный SET (PRE = LOW → Q = 1, /Q = 0).
3. Проверяется запись 1 (D = HIGH, CLK = HIGH → Q = 1, /Q = 0).
4. Проверяется запись 0 (D = LOW, CLK = HIGH → Q = 0, /Q = 1).
5. Проверяется инверсия выходов (Q ≠ /Q).

**Для JK-триггеров (74HC109):**

1. Проверяется асинхронный RESET (CLR = LOW → Q = 0, /Q = 1).
2. Проверяется асинхронный SET (PRE = LOW → Q = 1, /Q = 0).
3. Проверяется режим J=1, K=0 (установка Q = 1 по фронту CLK).
4. Проверяется режим J=0, K=1 (сброс Q = 0 по фронту CLK).
5. Проверяется режим TOGGLE (J=1, K=1 → Q меняется на противоположный).
6. Проверяется инверсия выходов (Q ≠ /Q).

## Как добавить новый триггер
1. Написать функцию тестирования одного триггера (аналог test_d_flipflop() или test_jk_flipflop()).
2. Написать функцию тестирования всей микросхемы (аналог test_74HC74()).
3. В handleCommand() добавить вызов.

## Как прошить ESP32

1. Открыть проект в Arduino IDE.
2. Выбрать плату ESP32 Dev Module.
3. Нажать Verify или Upload.
4. Чтобы получить бинарный файл:

``` Sketch -> Export compiled binary ```

Файл .bin появится в папке проекта.

## Ограничения

- GPIO34 и GPIO35 нельзя использовать как выходы (только входы).
- GPIO32, GPIO33 имеют ограничения при использовании ADC.
- Для микросхем с нестандартной топологией (например, 74HC02) может потребоваться отдельная карта PINx → GPIO.
- D-триггеры тестируются с задержками 5-10 мс для надежного срабатывания.
- JK-триггеры требуют активного LOW на входах /K и /PRE, /CLR.

## Назначение проекта

Этот тестер позволяет:

- проверять исправность логических микросхем 74HCxx
- тестировать D-триггеры (74HC74)
- тестировать JK-триггеры (74HC109)
- определять неисправные элементы внутри микросхемы
- использовать ESP32 как простой логический анализатор
- быстро диагностировать цифровые схемы
- расширять функционал под новые типы микросхем

## Схема подключения

Для 14-выводных микросхем (HC00, HC02, HC08, HC32):

- Вывод 7 (GND) → GND ESP32
- Вывод 14 (VCC) → 5V/3.3V ESP32
- Выводы 1-13 → согласно массиву PIN[]
  
Для 16-выводных микросхем (HC74, HC109):

- Вывод 8 (GND) → GND ESP32
- Вывод 16 (VCC) → 5V/3.3V ESP32
- Выводы 1-15 → согласно массиву PIN16[]
  
Индикация:

- GREEN_PIN (GPIO33) → OK (мигает 1 раз)
- RED_PIN (GPIO32) → FAIL (мигает 1 раз)
